# Contributing to OffroadTerrain AI

We welcome contributions! Please follow these guidelines to maintain project quality.

## Development Workflow
1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/amazing-feature`).
3. Implement your changes following PEP8 and using Type Hints.
4. Add unit tests in the `tests/` directory.
5. Ensure all tests pass (`pytest tests/`).
6. Submit a Pull Request.

## Coding Standards
- **Linter**: We use `flake8` for style checking.
- **Formatter**: Use `black` and `isort`.
- **Typing**: All new functions must have `mypy` compatible type hints.
- **Documentation**: Use Google-style docstrings.

## Pull Request Process
- Ensure the PR description clearly states the "Why" and "How".
- Address any reviewer feedback promptly.
- Maintain a clean commit history.
